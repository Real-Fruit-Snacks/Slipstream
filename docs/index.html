FEATURES

SSH with ! commands

Intercepts ! commands for tunnels, transfers, mapping, and logging. Everything else passes through to SSH unchanged.

Tunnel Management

iptables-style syntax for SSH tunnels. SOCKS proxy, local forward, reverse forward. Save and restore per target. Real forwarding via ssh -O forward.

$ !tunnel add --type socks -p 1080
$ !tunnel add --type local -s 8080 -d 10.10.10.50 -p 80
$ !tunnel list
$ !tunnel save
4
Transfer Methods

File Transfer

Automatic fallback chain: SFTP → SCP → cat-over-SSH → base64. Windows paths converted transparently. No separate SCP session needed.

Filesystem Mapper

Passive output parsing. Run ls, dir, find — the mapper builds a searchable map without sending extra commands. Pattern search, SUID finder, JSON export.

Session Logging

Every command gets its own timestamped log file. Session index tracks what you ran and when. Built for OSCP exam proof and engagement reporting.

2
Platforms

Target Identity

Targets identified by SSH host key fingerprint, not IP. Reconnect after DHCP changes. Access dual-homed hosts. Reuse lab IPs. Conflict detection with Archive/Keep/Ignore prompts.

$ slipstream ssh admin@10.10.10.5
fingerprint SHA256:xK9m2... → known target "DC01"
restoring 3 saved tunnels...

OS Detection

Auto-detects Linux vs Windows from SSH output. Adapts mapper parsers, CWD tracking, transfer paths, and loot targets per platform.

ARCHITECTURE

PTY-based interception

Wraps the real ssh binary. ControlMaster socket for multiplexing. Prompt-aware input routing.

1

PTY Layer

Spawns real ssh binary under a PTY. All SSH features pass through unchanged. SIGWINCH/SIGHUP/SIGTERM forwarded.

2

Input Router

Prompt-aware interception. Cooked mode buffers and checks for ! commands. Raw mode (vim, top) passes everything through.

3

Master Socket

ControlMaster=auto with Slipstream-owned socket. Tunnels via ssh -O forward. Transfers over the same connection.

4

Storage Layer

Per-target data keyed by fingerprint: target.toml, notes.txt, map.json, timestamped log directories.

GETTING STARTED

Build and connect

build.sh
$ git clone https://github.com/Real-Fruit-Snacks/Slipstream.git
$ cd Slipstream && cargo build --release

$ ./target/release/slipstream ssh user@10.10.10.5
connected — all SSH flags work normally
session.sh
# Inside the session
$ !tunnel add --type socks -p 1080
$ !upload linpeas.sh /tmp/
$ !map find suid
$ !loot
$ !note This is the domain controller
